{# The polling-aware wrapper around the work-queue chips + body. Every 5 seconds HTMX re-fetches `this_url` (the current filter + page + size) and outerHTML-swaps the entire pane, so: - newly-enqueued / state-changed work items appear without a reload, - the type / state chip counts refresh in lockstep with the body, - the wrapper's own `hx-get` is replaced too, so pagination and chip clicks (which also target #queue-pane) leave behind a fresh polling URL. `hx-swap="outerHTML"` is what keeps this loop self-sustaining; with `innerHTML` the wrapper's attributes wouldn't refresh and the URL would drift after a pagination click. `hx-headers` echoes the queue's current version back on every poll. The handler compares it to the live `queue_version` and returns 204 No Content when they match — HTMX then skips the swap and the server skips the template render. The version itself moves whenever any row in the filtered set is enqueued, transitions state, or is claimed/completed. See core_db.queue_version + core/ui.py queue(). #}
